portal v0.3.0 turns the clipboard-paste + port-forwarding tool into a
platform: the core is now a daemon exposing a local control API, with remote
exec (full PTY), pluggable SSH transports, and a public Go + TypeScript
surface for building your own "local shell, remote brain" apps.

This is also the **first release published on GitHub** — earlier releases
lived on the old GitLab project.

## New: `portal exec` — remote commands and interactive shells

Run commands on the dev box over the connection the daemon already maintains:
no extra handshake, faithful exit codes, streamed stdin/stdout/stderr.

```sh
portal exec -- make test     # exit code and output stream through faithfully
portal exec                  # interactive shell on the box, under a real PTY
portal exec -t -- htop       # force a PTY for a specific command
```

Interactive sessions get a full pseudo-terminal: raw mode, window-resize
(SIGWINCH) propagation, job control, and clean terminal restore even when the
client is killed by a signal. Every session is recorded (argv, uid, exit code,
duration, session id) in the append-only audit log, and the whole capability
can be disabled with `portal features exec off`.

## New: pluggable transports

`portal transport [system|native]` selects how portal reaches the box:

- **system** (default) — your system `ssh` over a ControlMaster multiplexed
  connection, using your existing setup exactly as-is.
- **native** — a built-in pure-Go SSH client: resolves `~/.ssh/config` (via
  `ssh -G`), dials `ProxyJump`/`ProxyCommand` chains itself, strict
  `known_hosts` checking, no `ssh` subprocesses.

Both (plus a `localexec` transport for same-machine development) implement one
transport contract, verified by a shared conformance suite.

## New: local control API + platform surface

- The daemon serves a versioned control API on an owner-only Unix socket
  (`~/.config/portal/api.sock`, peer-credential-gated): status, ports, feature
  gates, and `/v1/exec` (streamed exec with optional PTY). Schemas in
  `internal/localapi/openapi.yaml`.
- The Mac↔box wire protocol is specified in `docs/wire.cddl` with golden
  vectors under `docs/vectors/`, verified from both Go and TypeScript.
- Public Go packages under `pkg/` — `protocol`, `transport` (+`sshctl`,
  `sshnative`, `localexec`, conformance, `ptyx`), `run`, `client`, `agent`,
  `agentclient`, and more — with a test-enforced rule that `pkg/` never
  imports `internal/`.
- `clients/ts` is a zero-runtime-dependency TypeScript client, and
  `examples/shell-electron` drives a remote PTY session through it.

## Improvements

- `portal features` shows and toggles the capability gates (clip-image,
  clip-text, notify, exec) from the CLI; the daemon picks changes up live.
- Newly allowed ports now begin forwarding in ~0.5s (previously up to 60s).
- `portal <command> --help` shows each command's own flags instead of the
  top-level reference.
- Exec sessions carry a session id in the audit log so concurrent open/close
  lines pair up.

## Install / upgrade

```sh
curl -fL -o portal-darwin-arm64 \
  https://github.com/VikashLoomba/Portal/releases/latest/download/portal-darwin-arm64
chmod +x portal-darwin-arm64
./portal-darwin-arm64 install <ssh-host>
```

Upgrading from v0.2.0: run the same `install` — it replaces the binary in
`~/.local/bin`, redeploys the dev-box agent and shims, and re-runs the
`portal doctor` self-test. Apple Silicon (arm64) Macs only.
